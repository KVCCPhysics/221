Finally, I will give a quick description of other languages you may have seen. Please be aware that the below opinions are my own, and you may find others who disagree…

Numeric programming languages

In this course, we will use Python (and a bit of SageMath) to perform our analysis. There are many other languages you may see and might need to learn in your professional life. I will give a very rough (and perhaps opinionated) rundown of some popular languages you may encounter. Different professions will require different software, but I have identified 4 languages for numerics which are used heavily in engineering and science.

Python

High level general purpose language which is currently gaining in popularity.

  • Common Uses
    • Jack of all trades language used in many technical, business and tech fields
    • Scripting (small programs which perform a simple computer task)
    • Science (especially physics and chemistry)
    • Machine learning and data science (tech companies like Google etc)
    • Robotics and Mechatronics
  • Pros
    • Easy to create short programs for tedious calculations (scripting)
    • Fast development (less time coding)
    • Vast external libraries for almost all computing problems
  • Cons
    • Slow code execution
    • Steeper learning curve than Matlab or VBA (This decreases with experience)
    • Not widely used in engineering (This may be changing)
    • Requires installing software (but the software is free!)

C/C++

Low level general purpose language which is at the foundation of many softwares (including entire operating systems and large analysis softwares)

  • Common Uses
    • High performance codes which require lots of computation
    • Large projects with mix of software and hardware professionals
    • Embedded controllers (arduino)
  • Pros
    • FAST! Almost as fast as assembly
    • Well understood
    • Supported everywhere
  • Cons
    • Difficult to write code
    • Simple tasks require lots of boilerplate code
    • Requires code design before starting the project (bad for quick prototypes)
    • Requires installing software (but the software is free!)
    • Very steep initial learning curve

Matlab

High level language sold by MathWorks aimed as a numerical programming language for engineering. The syntax is especially convenient for matrix mathematics (think vector math). MatLab is highly recommended by university engineering instructors, but is losing ground in industry because of the cost (Python is FREE).

  • Common Uses
    • Used mostly in mechanical engineering
    • Not heavily used outside of engineering (this is bad…)
  • Pros
    • Matrix operations are easy to perform
    • Includes plotting libraries
  • Cons
    • Proprietary and EXPENSIVE (Very important)
    • Almost zero use outside of engineering
    • Missing key features of modern programming languages (no namespaces, weak object orientation)

Visual Basic (VB .NET)

Simple language which is natively supported in Microsoft Excel. Business language which is embraced by some engineering firms (not used at all in sciences).

  • Common Uses
    • Quick and dirty calculations which are used by non-engineers in a company
    • Analysis which you want to run directly in Excel
  • Pros
    • Included on almost all windows computers
    • Performs calculations inside of excel, which may be where the data is stored for the company.
  • Cons
    • Very shallow language
    • No visualization (can leverage excel for this)
    • No included packages for specialized fields

Symbolic (mathematics) programming languages

For those of you who enjoy mathematical manipulation, there is also another class of programming language which can be used to do symbolic manipulations. For example, in these languages, the constant Pi is not 3.1415926; it is the irrational number Pi and the software will treat it this way. You will not be required to learn any symbolic programming in this class, but I will use Sagemath for some example calculations

Sagemath

Programming suite which combines many free software packages. Not used heavily in industry.

  • Free software
  • built on top of python, so python code will run inside of SageMath
  • Disjointed language which can be confusing
  • Very new and has growing pains

Mathematica

Very powerful symbolic computation language. Backed by a very large corporation (Wolfram). This language is used in technical data analysis and research exploration for mathematics heavy fields.

  • Proprietary and EXPENSIVE!
  • Many mathematical routines come prepackaged
  • Best documentation of any programming language
  • Difficult to write high performance code

Maple

Competitor to Mathematica, but losing the battle. Heavily used in some specific branches of mathematics

  • Proprietary and EXPENSIVE!
  • Quick creation of plots and visualization
  • Used by WMU math department
  • Not heavily used by many industries outside of university

Other languages you may see

  • R - Statistics language which is similar to python. Python is starting to absorb many of its users.
  • Fortran - Older language used for legacy simulations. Superseded by C and C++. Used largely in large simulation suites which were written decades ago, and are still useful currently.
  • C# - Higher level extension to C++. Heavily used in windows and for developing video games. Successor to C++ in many non-science fields (including engineering).
  • Java - Enterprise programming language which is widely used in many business environments. Famous for its use of a “virtual” environment which supposedly can be run on any computer. Slow and clunky development. Rarely used for scientific or data heavy applications.
  • SQL - Standard language for storing related data in a database. Not as much of a programming language as it is a set of commands to get/set/manipulate large amounts of data.
  • Rust - Rust is intended to be a C++ replacement that places importance on correctness and memory safety. Rust programs are intended to never have segfaults.
  • Go - New language which is gaining popularity which borrows much of its syntax from C, but adds many convenient features like garbage collection. Go sets itself apart by using concurrency to do away with parallelism.
  • The list goes on and on…

Web languages (not used heavily for data analysis)

  • HTML - Static website markup language for displaying web page content. Widely used by web browsers to display static content.
  • Javascript - Dynamic website scripting language which is understood by a web browser (chrome, firefox, etc.). Used to make fancy things happen in your web browser, like moving images or dynamic content, but suffers from poor language design.
  • Ruby - High level web language used for web applications. It is more sane than Javascript, but cannot be interpreted by the web browser
  • PHP - Web scripting language. Fairly old language which is still heavily used by web developers.

Jupyter (or CoLab for Google)

Python is an interpreted language, which means you can run code as you go (no compilation). This means you can use it much like a glorified calculator. Another piece of software named jupyter is commonly used to make a web interface which can use the python interpreter right in your web browser.
